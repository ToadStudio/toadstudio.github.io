细心的读者可能发现了，这个printf()函数前面怎么多了个std::呢？

这个std::就代表使用std这个命名空间下的代码（注：std是英语standard（标准）的简写，是C++标准库的命名空间名称），和这个CPP文件里的代码完全不相干，所以程序可以编译通过，输出结果将会是1。

当然，本蒟蒻把“标准”一词标记成重点的原因，代表不是所有的编译器都遵守这个标准。有的编译器即使是使用了cstdio这个头文件也不区分命名空间，这其实并不符合标准。而上文所说的stdio.h与sctdio的区别，也正在与有无命名空间。所有C++的标准头文件内容都声明在了std命名空间里，这样来保证不会与代码源文件冲突。诸如此类的还有cstring和string.h，cstdlib和stdlib.h等等，总之前面带c字头而且不带.h后缀的就是标准的C++标准库，当然诸如iostream、fstream一类的也属于根正苗红的C++标准库头文件。写程序时尽量使用标准库，而不使用旧的头文件。

但是，每次使用标准库声明的变量/函数/类的时候，如果都这么std::一下，迟早要累死人。有一个最懒人的解决方法，也就是加上一行using namespace std（注：此处的std可以更换为其他的命名空间名字，视使用需求而定。只不过C++的标准库默认命名空间为std，所以才这里这么写。下同），相当于是把std命名空间下的所有声明移到当前的代码里面来，就不用每次使用标准库的东西的时候都在前面写一句std::了。但这样就会回到C的那个例子上面去。如果使用这种方法，虽然简单省事，但必须保证当前文件里面绝不能有任何东西可以和std命名空间里的东西冲突，否则就会报错。

输入输出函数

在iostream（标准流输入输出）头文件中，输入输出使用cin、cout配合流输入输出控制符（<<和>>）来实现。

Sample #1

cin函数，即流输入函数，配合流输入控制符（>>）来实现输入，使用方法如同cin >> var1 >> var2 >> ... >> varN。后面的变量个数以及数据类型随意，只要用>>隔开就行。但参数一定要是变量，不能是任何类型的常量或者数组（由变量组成数组的某一元素视为变量而非数组，下同）。至于什么是数组什么是变量什么是常量，下一篇再说。

cout函数，即流输出函数，配合流输出控制符（<<）来实现输出，使用方法如同cout << pragma1 << pragma 2 << ... << pragmaN。后面的参数个数以及数据类型随意，只要用<<隔开就行。参数可以是除了数组外的任意常量/变量。可以使用endl这一个IO操作符（名字很高大上，原理其实暂时无所谓，只要知道用处就好）来换行。endl视为一个参数。（注：endl也是iostream包含的内容，所以也在std命名空间里面）

顺便说一点，在C++以及其他很多很多很多语言里面，都是使用半角双引号（" "）来定义字符串（也就是一种包含很多字符的数据类型），使用半角单引号（' '）定义字符（也就是一种只包含单个字符的数据类型）。把Hello World!这句话用" "括起来，是因为这句话是一个字符串。如果不加上这个双引号，就会被识别为叫Hello和World!的两个变量名。

用iostream的函数很简单易懂，唯一要注意的就是不要混淆了<<和>>两种控制符。

在cstdio（或者stdio.h）中，使用printf()和scanf()两个函数

Sample #2

代码瞬间比起iostream的写法复杂了好多。

scanf()函数（输入函数）需要至少两个参数，第一个是一个字符串，包含要输入的变量的数据类型（也就是这个%d，表示一个32位整数（也就是int））。数据类型可以跟很多个，比如"%d%d%d"，但必须包含在同一个字符串以内，而且后面必须跟同样数目的按顺序排列的变量（比如在"%d%d%d"这个例子里面，就要跟着三个int类型的变量），使用逗号隔开，如同scanf("%d%d%d", &a, &b, &c)这样的格式。这样就会顺序读取三个值而且依次赋值给a、b、c三个变量。

除了%d，还有其他的数据类型。表格如下。

格式 字符意义
%d 十进制整数
%o 八进制整数
%x 十六进制整数
%u 无符号十进制整数
%f或%e 实型数(用小数形式或指数形式)
%c 单个字符
%s 字符串

现在看不懂没关系，之后就明白了

至于变量名字前面加的&符号，谓之曰【取地址符】，也就是去获取这个符号后面跟的那个变量的内存地址的。至于为什么要这么写，其实本蒟蒻也不知道，总之在scanf()里面，除了字符串（对应%s）以外的所有数据类型都要加上这样一个取地址符。

printf()函数（输出函数）也差不多，上面那个表格同样适用这个函数，但有一个不同的地方在于printf()可以出现这样的写法：printf("Hello World!")，而不用像scanf()一样地写printf("%s", "Hello World!")（虽然这么写也没错）。如果需要插入参数，可以在第一个字符串内写上%d或者%f之类的东西，后面再把参数写上。示例：printf("a is %d, b is %f", a, b)，这样%d就会被a的值替换掉，%f就会被b的值替换掉（前提是数据类型匹配，否则会报错）。而且不需要再使用取地址符了。

cstdio里面并没有任何endl之类的东西存在。要换行，必须在字符串里面加上\n（反斜杠n）。比如printf("Hello World!\n")，就能够在结尾换一行。这种带反斜杠的字符都是功能性的，也就是反斜杠会被当成一个符号而不是字符连着后面的的字母一起被编译器处理掉，称为【转义字符】（只要是字符串都如此，iostream也不例外），如果要输出反斜杠，那就在反斜杠面前再加一个反斜杠（也就是\\）就可以输出一个反斜杠，而且后面的字母不会被当作转义字符。

转义字符除了\n之外还有很多。这个之后再说了。

cstdio真是相当麻烦，而且一点也不智能就连类型都不能自己判断，还得手动指定，指定错了还要报错，でしょう（是不是）？本蒟蒻也这么觉得。所以平时正常使用，本蒟蒻更倾向于iostream的流输入输出函数。但是，也正因为printf()如此麻烦，也具备一些优点，比如控制输出格式之类（此之谓【格式化输出】），用printf()会比cout简单很多很多倍。但是其实格式化输出不太常用，而且网络上的教程写的也很清晰明了，有兴趣的话，自己问Google去，本蒟蒻就不赘述了。
