这个std::就代表使用std这个命名空间下的代码（注：std是英语standard（标准）的简写，是C++标准库的命名空间名称），和这个CPP文件里的代码完全不相干，所以程序可以编译通过，输出结果将会是1。

但是，每次使用标准库声明的变量/函数/类的时候，如果都这么std::一下，迟早要累死人。有一个最懒人的解决方法，也就是加上一行using namespace std（注：此处的std可以更换为其他的命名空间名字，视使用需求而定。只不过C++的标准库默认命名空间为std，所以才这里这么写。下同），相当于是把std命名空间下的所有声明移到当前的代码里面来，就不用每次使用标准库的东西的时候都在前面写一句std::了。但这样就会回到C的那个例子上面去。如果使用这种方法，虽然简单省事，但必须保证当前文件里面绝不能有任何东西可以和std命名空间里的东西冲突，否则就会报错。

Sample #1

cin函数，即流输入函数，配合流输入控制符（>>）来实现输入，使用方法如同cin >> var1 >> var2 >> ... >> varN。后面的变量个数以及数据类型随意，只要用>>隔开就行。但参数一定要是变量，不能是任何类型的常量或者数组（由变量组成数组的某一元素视为变量而非数组，下同）。至于什么是数组什么是变量什么是常量，下一篇再说。

cout函数，即流输出函数，配合流输出控制符（<<）来实现输出，使用方法如同cout << pragma1 << pragma 2 << ... << pragmaN。后面的参数个数以及数据类型随意，只要用<<隔开就行。参数可以是除了数组外的任意常量/变量。可以使用endl这一个IO操作符（名字很高大上，原理其实暂时无所谓，只要知道用处就好）来换行。endl视为一个参数。（注：endl也是iostream包含的内容，所以也在std命名空间里面）

顺便说一点，在C++以及其他很多很多很多语言里面，都是使用半角双引号（" "）来定义字符串（也就是一种包含很多字符的数据类型），使用半角单引号（' '）定义字符（也就是一种只包含单个字符的数据类型）。把Hello World!这句话用" "括起来，是因为这句话是一个字符串。如果不加上这个双引号，就会被识别为叫Hello和World!的两个变量名。

用iostream的函数很简单易懂，唯一要注意的就是不要混淆了<<和>>两种控制符。

scanf()函数（输入函数）需要至少两个参数，第一个是一个字符串，包含要输入的变量的数据类型（也就是这个%d，表示一个32位整数（也就是int））。数据类型可以跟很多个，比如"%d%d%d"，但必须包含在同一个字符串以内，而且后面必须跟同样数目的按顺序排列的变量（比如在"%d%d%d"这个例子里面，就要跟着三个int类型的变量），使用逗号隔开，如同scanf("%d%d%d", &a, &b, &c)这样的格式。这样就会顺序读取三个值而且依次赋值给a、b、c三个变量。

除了%d，还有其他的数据类型。表格如下。

格式 字符意义
%d 十进制整数
%o 八进制整数
%x 十六进制整数
%u 无符号十进制整数
%f或%e 实型数(用小数形式或指数形式)
%c 单个字符
%s 字符串

至于变量名字前面加的&符号，谓之曰【取地址符】，也就是去获取这个符号后面跟的那个变量的内存地址的。至于为什么要这么写，其实本蒟蒻也不知道，总之在scanf()里面，除了字符串（对应%s）以外的所有数据类型都要加上这样一个取地址符。

printf()函数（输出函数）也差不多，上面那个表格同样适用这个函数，但有一个不同的地方在于printf()可以出现这样的写法：printf("Hello World!")，而不用像scanf()一样地写printf("%s", "Hello World!")（虽然这么写也没错）。如果需要插入参数，可以在第一个字符串内写上%d或者%f之类的东西，后面再把参数写上。示例：printf("a is %d, b is %f", a, b)，这样%d就会被a的值替换掉，%f就会被b的值替换掉（前提是数据类型匹配，否则会报错）。而且不需要再使用取地址符了。

cstdio里面并没有任何endl之类的东西存在。要换行，必须在字符串里面加上\n（反斜杠n）。比如printf("Hello World!\n")，就能够在结尾换一行。这种带反斜杠的字符都是功能性的，也就是反斜杠会被当成一个符号而不是字符连着后面的的字母一起被编译器处理掉，称为【转义字符】（只要是字符串都如此，iostream也不例外），如果要输出反斜杠，那就在反斜杠面前再加一个反斜杠（也就是\\）就可以输出一个反斜杠，而且后面的字母不会被当作转义字符。

转义字符除了\n之外还有很多。这个之后再说了。

cstdio真是相当麻烦，而且一点也不智能就连类型都不能自己判断，还得手动指定，指定错了还要报错，でしょう（是不是）？本蒟蒻也这么觉得。所以平时正常使用，本蒟蒻更倾向于iostream的流输入输出函数。但是，也正因为printf()如此麻烦，也具备一些优点，比如控制输出格式之类（此之谓【格式化输出】），用printf()会比cout简单很多很多倍。但是其实格式化输出不太常用，而且网络上的教程写的也很清晰明了，有兴趣的话，自己问Google去，本蒟蒻就不赘述了。

C++的前世今生

Once upon a time，科技落后的人类终于掌握了最基本的电子计算机科技。当时的程序猿们只能使用能够直接让计算机读懂的二进制码来写程序。后来，出现了一个划时代的转折——汇编语言的诞生。从此以后，终于有了人类能够勉强阅读的代码。直到今天，汇编语言由于其与硬件的紧密联系仍然被应用于系统内核、工控、嵌入式计算一类的领域。但汇编语言仍然极度繁琐，由于其类似于调用CPU指令集的特性，导致代码通常很长而且不知所云。举个例子：

写这样一堆人类难以理解的话，只是为了在屏幕上打印一串Hello World!。如果要实现更复杂的功能，代码的复杂度则会呈指数上升，而且学习成本非常高，非常不容易上手。

于是，在1972年，贝尔实验室里面诞生了人类历史上计算机界的一块伟大的里程碑——C语言横空出世。

凭借着高效、灵活、功能丰富、表达力强和较高的可移植性等优点，C语言在短时间内占领了软件领域，直到现在，C语言仍被非常广泛地应用在软硬件行业里面。

由于C语言的成功，C语言的官方钦定后继者——C++语言，得以成功发布。C++语言采用了更为现代的理念（虽然比起现在的其他很多广泛应用的语言也相当远古了），支持了包括对后世影响深远的面向对象编程在内的新的编程范式，同时集成的SDL函数使广大程序猿们省下了不少代码，守护了千千万万挑灯夜战的年轻程序猿们脆弱的发际线。现在流行的编程语言中，99%都受到了C++的影响（数据是本蒟蒻胡扯的，别真信了，不过就这个现象而言是完全正确的），或多或少都能看出C++的影子。尽管C++也已经非常远古了，但其直到今天也完全没有过时的迹象，甚至仍然可以挤进最流行的编程语言排行榜的前几名。诸如上班族学生人人皆知的Microsoft Office的绝大部分代码、你现在正在盯着看的Windows/macOS/Linux/Android/iOS系统的绝大部分源代码、以及其他千千万万的程序、游戏引擎、类库，都使用C++写成。

那，是什么使得C++如此经久不衰呢？

C++的长处与短板

C++以其高效率、高兼容性、与底层硬件联系紧密闻名于世。从（稍微好一点的）单片机到树莓派到移动设备到ATM机到个人电脑到服务器工作站到NASA超算都具备运行C++的可能性，能比肩这样优秀的兼容性而且现在仍在使用的语言估计只有汇编和C了。前者应用率很低下，除了最最最最底层的东西基本没人用了，C则是C++的爸爸，本质上两者是一样的，所以经常可以看到C/C++这样的写法，因此就应用场合的广泛性而言，C家族的语言（狭义，只包含C/C++）的地位短期内不可撼动。C++的功能性与可扩展性都比C更强，运行性能却能基本持平C语言，而且图形编程到网络编程到嵌入式编程均可胜任，第三方类库也比C语言更丰富，可谓是上得厅堂下得厨房，所以在不那么贴近硬件底层的领域里面，C++的使用率碾压C语言。刚才提到的C++的运行效率可以比肩C语言，而C语言是目前人类常用的编程语言中除了汇编语言外绝对的效率No. 1，在这方面又有着不可替代的优势。业界笑话有云：

要实现同样的功能，Python要20s，Java要2s，C++只要1s

但是要实现同样的功能，C++要写200行，Java要100行，Python只要10行

这可谓是相当精辟了，不但道出了C++的高效率，同时也吐槽了C++的复杂性。

由于C++是古已有之的C的官方钦定升级版本，而且发布时间比起现在流行的任何一门编程语言都要早上许多，所以背负着极其沉重的历史包袱，C++标准一代代更新，这些历史包袱却无法卸掉。几个典型的问题使得很多人对C++望而却步：其复杂而且极易出错的指针和内存地址操作，和没有一样的字符串功能，以及（与现在的很多语言相比）极其低下的编码效率，等等。这些问题在现在看来不容忽视，所以即使不采用C++写程序，也无可厚非。但C++对当今计算机编程影响深远，学好C++，即便未来不用了，对于其他编程语言的学习也是大有裨益的。

关于C++的一些基本信息

（此列表大概会不定期更新的）

  1. C++是编译型语言，程序里不能有任何语法错误，否则无法通过编译过程，也就没有可执行文件。但是由于C++编译器太水了，一大半的逻辑错误和内存地址/空指针错误都分析不出来，所以即使是通过了编译也很有可能程序跑到一半遇到某个逻辑错误崩溃。这是十分常见的Bug，如果你对此不满，可以去了解一下Rust语言，可以做到在编译过程中指出一大半的逻辑错误。
  2. C++没有预编译过程，所以一律按从上到下的顺序执行代码，如果上面的代码调用了下面定义的东西（函数/变量名/etc.）就会报错。
  3. C++是强变量类型的语言，类型间的转换与各色魔法也是C++的乐趣之一
  4. 虽然C++有统一标准，但是细节上每种编译器和操作系统都会有不同的处理方法，所以C++并不是和Java一样，一旦写出来就放之四海皆准了，要针对不同的操作系统平台和编译器处理兼容性问题。在Linux平台上针对GCC编译器写的CPP文件不一定能通过Windows上的VC++的编译，反之亦然。即使都是GCC编译器，Linux平台上写的CPP文件也不一定能通过Windows的GCC移植版本MinGW-GCC的编译过程，反之亦然。即使都是Linux系统，针对Clang/LLVM编译器写的CPP文件也不一定能通过GCC编译器的编译过程，反之亦然。
  5. 每个版本的C++标准和功能都不同。C++98（1998年）或者C++03（2003年）是默认的C++标准，但更常用的是C++11（2011年）、C++14（2014年）和C++17（2017年）标准。目前最新的C++20（2020年）标准也快要发布了。理论上标准越新功能越多，实际上并不总是这样，标准与标准之间还有很多区别，这里不赘述了。总之根据自己的需求选择C++标准就好。
  6. 最标准的C++编译器是GNU组织发布的开源编译器GCC，仅支持Unix-like系统（Linux/macOS）的运行环境，但是Windows上面可以使用MinGW-GCC这一移植版本。推荐新手使用Linux + GCC的组合，是最原汁原味的C++编译与运行环境。GCC有很多版本，越新的版本能支持越新的标准，但老版本历经岁月磨砺稳定性通常会更好，此处可以根据自己的侧重点选择。本蒟蒻喜欢追求新事物，不太在乎稳定性，所以生产环境永远采用GCC最新版本，但在侧重稳定性的服务器上则有可能采用GCC 6甚至GCC 4这样的远古版本。

写作动机

很多时候，会有一些一闪而过的念头，或者学习了一些东西，却懒得打开笔记本。很多时候，发现了一些崭新的观点，却碍于笔尖与文思，迟迟得不到记录。

本蒟蒻曾经在Twitter上写一些话，试图达到在这样的目的。然而Twitter的200字限制导致本蒟蒻只能不断地删减，删减，继续删减。最后，饱满的思想被剔得只剩下干瘪的骨架。也鲜有人愿意在Twitter这样一个显然快餐化的地方，查看长篇大论的文字。久而久之，本蒟蒻也没了写下去的念头。

然而，有那么一个博客，编程随想的博客，启发了本蒟蒻，让本蒟蒻受益良多。从他的字句中，本蒟蒻读出了无数闪着光芒的深邃思想，有如大海般的深刻与洞察。在Twitter上早已不可寻觅的愉快感受——与真正的智慧相伴的感受，终于回到了本蒟蒻的心间。

这正是博客的好处——可以深入的记录某一方面的内容，不必绞尽脑汁的压缩文字，不必为了涨Fo而曲意逢迎。字里行间里，流露着作者真实的思想与感受，闪烁着理性与思考的光芒。在写作之时，可以心平气和，对着Visual Studio Code的黑色UI，写下一点什么，或者什么也不写。这亦是Micro Blog平台所不能企及的优势。

以上便是是本蒟蒻开始写博客的契机。
